记录一下使用github搭建个人博客网站

2023-01-11
学习
  • giscus
  • hexo
N 人看过

前言

临近过年，手上也没有什么新的项目，闲来无事就想着搭一个博客网站来玩玩。其实这也不是我第一次搭了，说多了都是泪😂，都是搭起来的时侯写两篇，然后服务器就放起吃灰，钱也花了，效果还没到达。最终还是决定要白嫖~~~~

准备工作

我对博客网站的搭建就仨要求：①搭建简单 ②免费 ③可扩展。

满足以上三个要求的可选方案有很多，但是我最终选用 github + hexo 的方式搭一个静态的blog网站。目前在我看来它俩最大的优点就是都可以白嫖，写一篇文章你就要手动提交一次到github上，当然你也可以一次提交几篇，如果你是生产队的牛的话~~~ ~ 只要不是频繁的提交、修改勉强能接受吧。

github大家应该都不陌生，hexo的话在我看来就是一个帮你搭建一个静态博客页面的脚手架。你说hexo咋安装？客官麻烦出门左转找度娘~~~~

图片插入

在写文章的过程中，图片的插入应该是很频繁的事情，怎么能让发布在网上的文章中的图片能显示出来。有两种方案摆在我面前：

①将图片存在hexo的资源文件夹下，文章中使用相对路径来访问这些图片。

出现问题

但是值得注意的是，使用该方式引入图片会导致一个问题，在首页能看见上图☝️，但是在点击进入文章详情后上图却不能显示。当时我大概猜想了一下可能是文章的存储的路径发生了变化，也就是说首页看见的文章和进入文章详情看见的文章可能并不是同一篇文章（虽然文章内容一致，但实际存储路径不一致)不然不会存在图片在另一边显示不正确的问题。

原因

如何验证以上猜想，那就要去查看一下通过执行hexo一键部署命令hexo deploy后生成的静态页面文件结构是怎样的了，通过查看上传到gitHub仓库的页面文件见下图👇

②使用图床的方式引入图片

这种方式就是将你要展示的图片上传到你的资源服务器上，由服务器返回一个资源对应的url，你就可以使用Markdown的标准语法来插入图片。这也是我较推荐的一种方式，因为你也可以将github作为你的图床，这就意味这又可以白嫖😆~~~~这里强烈推荐一款图床插件PicGo再配合Typora使用，当你要在文章中插入图片时简直不要太舒服，孩子用了都说好。

Hexo主题

hexo提供的主题也是非常丰富，当然这也非常感谢那些提供精美主题的开源作者，我一眼就相中了一款名为Aomori的主题，也许是作者希望使用者多关注关注他写的代码，提供的文档不是太全，算一个小坑吧。但也不得不说是一个很棒的主题。

评论区

评论区应该是一个博客锦上添花功能吧，为什么我说Aomori是一个很棒的主题呢，以为它还集成了一些评论插件的功能，我选用的是Giscus作为我的评论系统，值得留意的是这个评论系统是由GitHub Discussions驱动的，它居然把github提供的项目的评论区功能给单独抽离出来使用，非常佩服Giscus的作者👍。gitHub！！！我哭死😭，为我们这些白嫖个人博客网站的用户考虑的多周到呀😏

结束

到此为止一个基于github搭建的个人博客网站就正式搭建完成，目前该网站还未提交给搜索引擎收录，知道自己菜，文章没啥营养，但作为技术菜鸟也有自己的理想，后续会发布一些技术相关的文章，算是一点一点记录一下自己的足迹。
